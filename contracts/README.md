# Contracts

Solidity smart contracts for the NoirLimit poker protocol. Built with [Foundry](https://book.getfoundry.sh/).

## Directory Structure

```
contracts/
├── poker/              # Core poker game contracts
│   ├── PokerTable.sol  # Main game contract - manages table state, player joins, hand lifecycle
│   ├── GameState.sol   # Game state machine - tracks phases (pre-flop, flop, turn, river, showdown)
│   └── HandEvaluator.sol # On-chain hand ranking logic for showdown resolution
│
├── spectator/          # Spectator wagering market
│   ├── SpectatorMarket.sol  # Wager placement and resolution
│   └── OddsEngine.sol       # Odds calculation based on public game state
│
├── verifiers/          # Auto-generated Noir proof verifier contracts
│   ├── ShuffleVerifier.sol  # Verifies deck shuffle proofs
│   ├── DealVerifier.sol     # Verifies card dealing proofs
│   ├── BetVerifier.sol      # Verifies bet validity proofs
│   └── RevealVerifier.sol   # Verifies card reveal proofs at showdown
│
├── interfaces/         # Contract interfaces
│   ├── IPokerTable.sol      # Poker table interface
│   ├── ISpectatorMarket.sol # Spectator market interface
│   └── IVerifier.sol        # Common verifier interface
│
└── libraries/          # Shared utility libraries
    ├── CardLib.sol          # Card encoding/decoding utilities
    └── BettingLib.sol       # Bet validation and pot calculation helpers
```

## Architecture

### Poker Table (`poker/`)

The `PokerTable.sol` contract is the main entry point. It handles:
- Player registration and buy-in
- Hand lifecycle management (initiate shuffle -> deal -> betting rounds -> showdown -> payout)
- Proof submission and verification delegation
- Pot management and payout distribution

`GameState.sol` manages the state machine transitions:
```
WAITING -> SHUFFLE -> DEAL_PREFLOP -> BET_PREFLOP -> DEAL_FLOP -> BET_FLOP ->
DEAL_TURN -> BET_TURN -> DEAL_RIVER -> BET_RIVER -> SHOWDOWN -> PAYOUT
```

`HandEvaluator.sol` implements Texas Hold'em hand ranking to determine winners at showdown.

### Spectator Market (`spectator/`)

A separate contract that reads from `PokerTable` to let spectators place wagers on game outcomes. Wagers resolve automatically when the hand completes.

### Verifiers (`verifiers/`)

These are auto-generated by Noir's Solidity verifier export (`nargo codegen-verifier`). Each circuit gets its own verifier contract. **Do not edit these manually** -- they get regenerated from the circuits.

### Interfaces (`interfaces/`)

Standard Solidity interfaces for cross-contract calls and frontend integration.

### Libraries (`libraries/`)

Shared utilities:
- `CardLib.sol` - Encodes cards as uint8 (4 bits suit, 4 bits rank), provides helpers for commitment schemes
- `BettingLib.sol` - Validates bet amounts, calculates side pots, manages blind structure

## Build & Test

```bash
# Build
forge build

# Test
forge test

# Deploy to testnet
forge script scripts/Deploy.s.sol --rpc-url $RPC_URL --broadcast
```

## Key Design Decisions

- **Verifier contracts are separate** rather than inlined, so they can be upgraded independently if circuits change
- **Game state lives on-chain** but card values are stored as commitments (hashes) until reveal
- **Spectator market is a separate contract** to isolate wager logic from game logic
